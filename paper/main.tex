\begin{abstract}
% STUB: refine after §7 drafting (§7 == post-skeleton prose pass).
% Pasted verbatim from outline v2.1 abstract; H1 sentence carries the
% verified 95% bootstrap CI [+0.08, +0.92] excluding zero.
Frontier LLMs are increasingly applied to code reasoning, yet whether the
\emph{format} of input code affects reasoning quality, independent of the
\emph{information content} of the representation, is rarely tested with
controls. We present a methodology for such testing --- a 5-condition design
with an information-parity control (C1+) projecting a contract-augmented IR
back into Python source --- and instantiate it on T2 bug detection across 30
retained non-recursive Python functions, two confirmatory frontier LLMs, and
a successor pair (post-confirmatory exploratory replication). The primary
endpoint H1 (C4 $>$ C1+) yields a moderate positive effect (rank-biserial
$r = +0.58$, 95\% CI $[+0.08, +0.92]$ excludes zero) but does not reach
Holm-corrected significance under the pre-registered 9-test family
(corrected $p = 0.205$). Exploratory analyses surface a directional pattern:
H3 (in-text information enrichment helps) inverts on all four model-runs
spanning two LLM generations. Post-hoc inspection reveals that the T2
composite score saturates near $2/3$, compressing the effective Wilcoxon $n$
from 30 to 13--14 and constituting the structural source of confirmatory
underpowering. We discuss implications for both the empirical claim and the
methodology, and propose rubric refinement and T1/T3 collection as the next
steps.
\end{abstract}

\section{Methodology}
\label{sec:3}

\subsection{Study phases}
\label{sec:3.0}
The study was conducted over a 90-day window (Constitution \S13) in
three phases.

\textbf{Phase~1 (days 1--25)} covered pipeline construction and dataset
curation. We implemented the representation transformers, the contract
generator, and the automated scoring scripts, and curated the 30
retained non-recursive Python functions. This phase produced the
conversion pipelines for C1 through C4 and the C4\,$\to$\,C1+
projection.

\textbf{Phase~2 (days 26--50)} was a pilot. A pilot run exercised a
small subset---approximately three functions across all five
conditions, all three tasks, and both models, on the order of ninety
items per run---to calibrate the runner and the automated scoring
scripts. Two pilot runs were executed; the second was clean. Pilot
data is not pooled with confirmatory data: the confirmatory analysis
derives solely from a fresh collection on the 30 retained functions
(\texttt{full\_t2\_confirmatory\_v2}).

\textbf{Phase~3 (days 51--90)} covered confirmatory data collection
(T2 only), a post-confirmatory exploratory replication on a successor
frontier model pair, and analysis.

One naming artifact warrants explicit note. The runner module is
\texttt{src/experiment/run\_pilot.py}, named during the Phase~2 pilot
and reused without modification for confirmatory data collection. The
name reflects implementation history, not methodological status: the
confirmatory run is not a pilot.

\subsection{Conditions and dataset}
\label{sec:3.1}

\subsubsection{The five conditions}
\label{sec:3.1.1}
Table~\ref{tab:conditions} summarizes the five experimental conditions.
The cumulative chain C1\,$\to$\,C2\,$\to$\,C3\,$\to$\,C4 progressively
enriches information, while the branch C1\,$\to$\,C1+ enriches
information in textual form. The central comparison is C4 versus C1+:
both carry the same type and contract annotations and differ only in
format---structured IR versus annotated Python source. This pairing
isolates the structure axis from the information axis.

\subsubsection{Dataset composition}
\label{sec:3.1.2}
The dataset consists of 30 retained non-recursive single Python
functions, manually curated within the 90-day scope. Their domain
distribution (Appendix~\ref{app:A}) is predominantly list and
array manipulation (20 of 30), with smaller representation from
descriptive statistics (6), string processing (2), arithmetic (1),
and control-flow or parser-like logic (1). This list-operations
predominance reflects the prevalence of such functions in
standard-library and algorithmic-task settings rather than
intentional curation; we treat it as an external-validity limitation
(\S\ref{sec:5.5}).

A static recursion-exclusion check at the curation stage rejects any
function with self-recursion or mutually recursive helpers; the
90-day scope restricts the study to non-recursive code
(Constitution \S17.5).

For each function we prepare ground-truth artifacts: a single buggy
variant whose defect falls into one of five categories (off-by-one,
wrong comparison, missing boundary, incorrect variable, swapped
arguments; Constitution \S17.3), an annotation of the bug location
and category, and a reference-fix test suite.

\subsection{Information parity: operational definition and construction}
\label{sec:3.2}
We define the \emph{information content} of a representation as the
set of typed propositions derivable from it: the function signature
types and the parameter and return contracts (precondition,
postcondition, invariant). Under this definition C1+ and C4 are
equivalent by construction.

C1+ is produced by projecting C4's annotations into Python source:
PEP~484 inline type hints in the signature, plus a structured
docstring block carrying the contract clauses in a fixed template.
The projection is implemented in
\texttt{src/pipeline/ir\_to\_annotated\_text.py}.

We acknowledge a residual asymmetry. The projection does not preserve
AST node-kind metadata, IR-level syntactic child-ordering, or explicit
cross-reference structure among contract clauses, which become
free-form docstring text. Under our operational definition these are
\emph{encoding}, not \emph{information}; a more demanding
definition---Shannon-style bit count, or recoverability of the source
IR from the projected text---would assign C4 strictly higher
information content. We do not adopt that definition, because the
empirical question is whether structure aids reasoning \emph{given the
same propositional content}, not whether representations differ in
arbitrary encoding bits. A round-trip validation measuring
C4\,$\to$\,C1+\,$\to$\,C4$'$ structural recovery is identified as
journal-version follow-on work (Appendix~\ref{app:D}).

Contracts for C4, and therefore for C1+, are generated by an LLM that
is not used as a test subject and are reviewed by the author under the
contract review protocol (Constitution \S17.2): the allowed edits are
factual correction, deletion of vacuously true clauses, and a single
missing-clause addition. Provenance---raw, reviewed, and diff---is
logged for every function.

\subsection{Tasks and primary endpoint}
\label{sec:3.3}
Three task types are defined: T1 (code understanding), T2 (bug
detection), and T3 (code transformation). The 90-day study collected
only T2; T1 and T3 are uncollected, and the six corresponding cells of
the confirmatory family are insufficient-data placeholders
(\S\ref{sec:3.5}, \S\ref{sec:4.2}).

T2 was pre-registered as the primary endpoint (Constitution \S17.4)
for objectivity: its 0--3 composite score---the sum of location,
diagnosis, and fix sub-scores---is fully automated and requires no
human judgment at scoring time. We honor that pre-registration
throughout \S\ref{sec:4}: H1 on T2 is the primary endpoint and
carries the strongest interpretive weight.

\subsection{Models and sampling regime}
\label{sec:3.4}
The confirmatory pair comprised \texttt{gpt-5.4} and
\texttt{claude-opus-4-6}, sampled at \texttt{temperature}\,$=$\,$0$.
A post-confirmatory exploratory replication used the same-provider
successor frontier pair---\texttt{gpt-5.5} and
\texttt{claude-opus-4-7}---sampled at provider-default temperature.
The successor models reject \texttt{temperature}\,$=$\,$0$ at the
API level; provider-default temperature is therefore a hard
constraint, not a methodological preference.

This introduces a sampling-regime asymmetry: deterministic on the
confirmatory pair, stochastic on the successor pair. We accordingly
frame the cross-pair comparison as a robustness check under
temperature relaxation, not as cross-generation replication; the two
pairs are not directly comparable under a shared sampling regime. The
successor pair was selected as the same-provider frontier successors
so that representation effect, rather than provider drift, remains
the axis of interest.

\subsection{Statistical plan}
\label{sec:3.5}
The pre-registered confirmatory family comprises three hypotheses
(H1: C4\,$>$\,C1+; H2: C4\,$>$\,C1; H3: C1+\,$>$\,C1) crossed with
three primary metrics (T1, T2, T3), giving nine tests. For each test
we compute, per function, a paired difference between the two
conditions after averaging the two model scores; we then apply the
Wilcoxon signed-rank test with a one-sided ``greater'' alternative
matching the pre-registered direction, take the rank-biserial
correlation $r = (T^{+} - T^{-})/(T^{+} + T^{-})$ as the effect size,
apply the Holm--Bonferroni step-down correction across the nine-test
family at $\alpha = 0.05$, and report a 95\% percentile bootstrap CI
on $r$ from 10{,}000 seeded resamples.

We honor $m = 9$ as pre-registered. Six of the nine cells are
insufficient-data placeholders because T1 and T3 were not collected
(\S\ref{sec:3.3}); in reporting we distinguish the three
collected-and-tested cells from the six scope-limited-uncollected
cells, but we do not retroactively reduce $m$ to 3. Post-hoc
$m$-reduction would forfeit the value of pre-registration: the cost
is reduced corrected power, the benefit is that the result speaks
under the rules committed to before data were seen. Effective sample
size per test is further compressed because Wilcoxon discards zero
paired differences: against $n_{\text{pairs}} = 30$, the tie
distribution induced by the rubric (\S\ref{sec:3.6}) leaves an
effective $n$ of 13--14.

Per-model analyses are exploratory by Constitution \S17.4 and are
reported uncorrected with explicit exploratory labeling.

On bootstrap CI provenance: Constitution \S17.4 mandates 95\%
bootstrap CIs on effect sizes. The frozen
\texttt{analyze\_confirmatory.py} does not compute CIs; CIs were
computed via a sibling implementation verified against the frozen
analyzer's point estimates. CI computation does not alter any
hypothesis-testing decision; it adds reporting fidelity. Confirmatory
CIs are reported in \S\ref{sec:4} alongside point estimates;
provenance is recorded in Appendix~\ref{app:E}.

\subsection{Score rubric and its empirical distribution}
\label{sec:3.6}
The T2 composite score is the integer sum of three binary
sub-scores: location (0/1), diagnosis (0/1), and fix (0/1). Per-model
support is therefore $\{0,1,2,3\}$; the per-function 2-model-averaged
support is $\{0.0, 0.5, 1.0, 1.5, 2.0, 2.5, 3.0\}$.

We surface the empirical distribution here---before results---because
it constrains what the analysis can detect. In the 300-record
confirmatory run the per-model raw composite distribution is $0$:
19~records (6\%), $1$: 89 (30\%), $2$: 192 (64\%), and $3$: 0 (0\%).
No model scored $3/3$ on any T2 item; the location sub-score is the
empirical bottleneck. After 2-model averaging, composite supports
$2.5$ and $3.0$ are both empty, leaving an effective range of
$[0.0, 2.0]$ with mode $2.0$. Across H1, H2, and H3 at least 53\% of
paired differences are exactly zero; Wilcoxon discards these, and
the effective $n$ contracts to 13--14 from $n_{\text{pairs}} = 30$.

The ceiling is a property of the as-implemented rubric, not of the
data. Expanding $n$ alone does not solve underpowering: so long as
the location sub-score saturates, additional functions produce ties
at the same rate. The path forward (\S\ref{sec:6}) is rubric
refinement, or supplementing the composite with a continuous-valued
T2 metric. Score histograms are shown in Figure~\ref{fig:f3}.

\section{Results}
\label{sec:4}

\subsection{Confirmatory: H1/T2 primary endpoint}
\label{sec:4.1}
The primary endpoint of the study is the pre-registered H1 test on
T2: under the C4-versus-C1+ comparison, does the structured-IR
condition outperform its information-parity textual control on
bug-detection composite score? Table~\ref{tab:h1-t2-primary} reports
the full result.

\textbf{Decision: fail to reject $H_0$ at $\alpha = 0.05$ corrected.}
Under Constitution \S9 success criteria this places the study at
minimum success---the experiment was completed, materials released,
and no confirmatory hypothesis was rejected.

The non-rejection is informative rather than empty. The point
estimate $r_{\text{rb}} = +0.581$ is moderate and lies in the
pre-registered direction; the 95\% percentile bootstrap CI
$[+0.077, +0.917]$ excludes zero; and the one-sided raw $p = 0.023$
would reject $H_0$ in isolation. The failure occurs at the
Holm--Bonferroni family-correction stage with $m = 9$: the corrected
$p = 0.205$, and at the present effective Wilcoxon $n = 14$
(compressed from $n_{\text{pairs}} = 30$ by the rubric-induced tie
distribution) the correction absorbs the moderate effect. The
structural source of the under-rejection---the score-rubric
ceiling---is documented in \S\ref{sec:3.6} and discussed in
\S\ref{sec:5.3}.

\subsection{Confirmatory: secondary T2 layer and family completeness}
\label{sec:4.2}
Table~\ref{tab:confirmatory-family} reports the full nine-test
confirmatory family: three hypotheses crossed with three primary
metrics, with $m = 9$ honored as pre-registered.

Of the three tested cells (all on T2), two yield rank-biserial $r$
consistent with their pre-registered direction: H1/T2 with
$r_{\text{rb}} = +0.581$ (CI $[+0.077, +0.917]$, raw $p = 0.023$)
and H2/T2 with $r_{\text{rb}} = +0.200$ (CI $[-0.333, +0.700]$, raw
$p = 0.233$). The third, H3/T2, yields $r_{\text{rb}} = -0.473$
(CI $[-0.885, +0.077]$, raw $p = 0.969$)---\emph{opposite} the
pre-registered direction. None of the three reaches Holm-corrected
significance. H1/T2 is the only tested cell whose 95\% CI excludes
zero; the H3/T2 upper bound ($+0.077$) only just crosses zero.

The remaining six cells of the family are T1 and T3 hypotheses;
T1 and T3 were uncollected within the 90-day scope
(\S\ref{sec:3.3}) and contribute Holm $p = 1.000$ by convention.
We do not retroactively reduce $m$. The distinction between
collected-and-tested and scope-limited-uncollected cells is a
reporting matter (\S\ref{sec:3.5}), not a multiplicity adjustment,
and the six uncollected cells remain in the family.

The H3/T2 directional inversion is the strongest exploratory signal
in the dataset; its per-model decomposition is taken up in
\S\ref{sec:4.4}.

\section{Discussion}
\label{sec:5}

\subsection{What the primary endpoint does and does not support}
\label{sec:5.1}
% STUB (REVISED IN v2.1: CI use): The H1/T2 non-rejection is not evidence
% of no effect. Point estimate r=+0.58 is moderate, AND the 95% bootstrap
% CI [+0.077, +0.917] EXCLUDES ZERO; the one-sided raw p=0.023 would
% reject in isolation; failure is at the family-correction stage with
% m=9, six of which are insufficient-data placeholders. Honest reading:
% data are CONSISTENT WITH the pre-registered direction at moderate
% effect size with a CI that excludes zero, and the current effective
% power (n≈14 after ties) does not support a confirmatory claim. Rubric
% refinement, not n alone, is the path forward.

\subsection{Cross-run direction concordance: what it permits and forbids}
\label{sec:5.2}
% STUB: H1 direction positive in all four model-runs; H3 direction
% negative in all four. While exploratory and not multiplicity-controlled,
% this pattern constrains the space of explanations: a single-model
% artifact for H1 is ruled out, and the H3 inversion is not noise on a
% single run. Exploratory signals do NOT promote H1 to confirmatory
% status, but they do shift the prior on what a properly-powered
% follow-up would find.

\subsection{The score-rubric ceiling is a structural finding, not a side issue}
\label{sec:5.3}
% STUB: The location sub-score's empirical near-saturation (§3.6) is the
% binding constraint on confirmatory power. This is a property of the
% as-implemented T2 rubric, not of the dataset or models. Scaling n from
% 30 to 50 or 100 would not change the saturation rate; it would only
% multiply tied paired differences. The follow-up should refine the
% location sub-score or supplement with a continuous T2 metric BEFORE
% expanding n. Methodological finding with re-use value beyond this study.

\subsection{Implications for the long-term Lumen vision}
\label{sec:5.4}
% STUB: Constitution v0.2.2 §3 specifies a reassessment threshold for
% the canonical-core-over-surface-text bet: an outcome of C1+≥C4 on all
% primary metrics across both confirmatory models would have triggered
% formal reassessment. The observed outcome — C4>C1+ in direction with
% moderate effect size and a CI excluding zero, failing Holm — does NOT
% trigger reassessment, nor confirmatory support. The bet's empirical
% motivation remains intact at the level of point estimates and weakened
% at the level of statistical confirmation; the project continues at the
% architectural level pending T1/T3 collection, rubric refinement, and a
% properly-powered confirmatory re-test.
