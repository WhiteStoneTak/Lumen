% Abstract
% =============================================================
\begin{abstract}
% STUB: refine after §7 drafting (§7 == post-skeleton prose pass).
% Pasted verbatim from outline v2.1 abstract; H1 sentence carries the
% verified 95% bootstrap CI [+0.08, +0.92] excluding zero.
Frontier LLMs are increasingly applied to code reasoning, yet whether the
\emph{format} of input code affects reasoning quality, independent of the
\emph{information content} of the representation, is rarely tested with
controls. We present a methodology for such testing --- a 5-condition design
with an information-parity control (C1+) projecting a contract-augmented IR
back into Python source --- and instantiate it on T2 bug detection across 30
retained non-recursive Python functions, two confirmatory frontier LLMs, and
a successor pair (post-confirmatory exploratory replication). The primary
endpoint H1 (C4 $>$ C1+) yields a moderate positive effect (rank-biserial
$r = +0.58$, 95\% CI $[+0.08, +0.92]$ excludes zero) but does not reach
Holm-corrected significance under the pre-registered 9-test family
(corrected $p = 0.205$). Exploratory analyses surface a directional pattern:
H3 (in-text information enrichment helps) inverts on all four model-runs
spanning two LLM generations. Post-hoc inspection reveals that the T2
composite score saturates near $2/3$, compressing the effective Wilcoxon $n$
from 30 to 13--14 and constituting the structural source of confirmatory
underpowering. We discuss implications for both the empirical claim and the
methodology, and propose rubric refinement and T1/T3 collection as the next
steps.
\end{abstract}

\label{sec:1.2}
% STUB: Naive comparisons (text vs AST) confound representation format with
% information content. Prior work either trains models on different
% representations (confounding format with training signal) or applies
% structured prompting heuristically without controls; no prior controlled
% study combines a frozen-frontier-LLM target, an information-parity control,
% and pre-registered family-wise correction.

\subsection{Contributions}
\label{sec:1.3}
% STUB: Three contributions: (1) an information-parity testing methodology
% with a 5-condition design and a C1+ parity control isolating the structure
% axis; (2) a pre-registered 9-test confirmatory family (3 hypotheses × 3
% metrics) under Holm-Bonferroni at α=0.05; (3) diagnostic findings from a
% 30-function pilot — direction-consistent but Holm-non-significant primary
% endpoint, exploratory H3 inversion across all four model-runs, and a

\subsection{The single-sentence proposition}
\label{sec:1.4}

\begin{quote}
We propose a controlled experimental methodology for measuring
code-representation effects on frozen frontier LLMs, instantiate it on bug
detection across 30 retained Python functions and two LLM generations, and
report a primary endpoint that is direction-consistent but underpowered,
alongside an exploratory directional inversion of the
in-text-information-enrichment hypothesis and a structural source of

\subsection{Information parity: operational definition and construction}
\label{sec:3.2}
% STUB: Information content := the set of typed propositions derivable from
% a representation (signature types, parameter/return contracts: pre, post,
% invariant). Under this definition, C1+ and C4 are equivalent by
% construction. C1+ is constructed by projecting C4's annotations into
% Python source: PEP 484 inline type hints in the signature plus a fixed
% structured-docstring template carrying the contract clauses
% (src/pipeline/ir_to_annotated_text.py). Residual asymmetry — AST
% node-kind metadata, syntactic child-ordering, and explicit
% cross-reference structure between contract clauses — is not "information"
% under our operational definition; a more demanding definition (Shannon
% bits, recoverability of the source IR) would assign C4 higher information
% than C1+. Round-trip validation (C4 → C1+ → C4') is journal-version
% follow-on work (Appendix D, \cref{app:D}). Contracts are LLM-generated by
% a non-test-subject model and reviewed by the author per Constitution
% §17.2 (allowed edits: factual correction, vacuous-clause deletion, single
% missing-clause addition); provenance is logged for every function.

\subsection{Statistical plan}
\label{sec:3.5}
% STUB: Pre-registered confirmatory family = 3 hypotheses × 3 primary
% metrics = 9 tests. H1: C4>C1+ (structure at constant information).
% H2: C4>C1 (total enrichment). H3: C1+>C1 (information enrichment in
% text). Per test: per-function paired difference (2-model averaged) →
% Wilcoxon signed-rank (one-sided "greater", zero_method='wilcox') →
% rank-biserial r = (T+ − T−)/(T+ + T−) on the same non-zero-diff set →
% Holm-Bonferroni at α=0.05 across the 9-test family → 95% percentile
% bootstrap CI (10,000 resamples, seed 20260427).
%
% On the correction: m=9 honored as pre-registered; six cells are
% insufficient-data placeholders (T1/T3 uncollected). We distinguish in
% reporting between collected-and-tested (3) and scope-limited-uncollected
% (6) without retroactive m-reduction. Effective n per test is 13–14
% (vs n_pairs=30) — the binding power constraint, see §3.6. Per-model
% analysis is exploratory (Constitution §17.4); per-model results in §4.3
% are uncorrected and labeled exploratory.
%
% On bootstrap CI provenance: Constitution §17.4 mandates 95% bootstrap CI
% on effect sizes. The frozen analyze_confirmatory.py does NOT compute CI;
% we compute CI via a sibling implementation
% (scratch/bootstrap_ci_confirmatory.py) verified against the frozen
% analyzer's point estimates. CI computation does not modify any
% pre-registered hypothesis-testing decision; it adds reporting fidelity.
% Confirmatory CIs reported in §4 alongside point estimates. (Provenance
% manifest: Appendix E, \cref{app:E}.)

\section{Discussion}
\label{sec:5}

\subsection{What the primary endpoint does and does not support}
\label{sec:5.1}
% STUB (REVISED IN v2.1: CI use): The H1/T2 non-rejection is not evidence
% of no effect. Point estimate r=+0.58 is moderate, AND the 95% bootstrap
% CI [+0.077, +0.917] EXCLUDES ZERO; the one-sided raw p=0.023 would
% reject in isolation; failure is at the family-correction stage with
% m=9, six of which are insufficient-data placeholders. Honest reading:
% data are CONSISTENT WITH the pre-registered direction at moderate
% effect size with a CI that excludes zero, and the current effective
% power (n≈14 after ties) does not support a confirmatory claim. Rubric
% refinement, not n alone, is the path forward.

\subsection{Cross-run direction concordance: what it permits and forbids}
\label{sec:5.2}
% STUB: H1 direction positive in all four model-runs; H3 direction
% negative in all four. While exploratory and not multiplicity-controlled,
% this pattern constrains the space of explanations: a single-model
% artifact for H1 is ruled out, and the H3 inversion is not noise on a
% single run. Exploratory signals do NOT promote H1 to confirmatory
% status, but they do shift the prior on what a properly-powered
% follow-up would find.

\subsection{The score-rubric ceiling is a structural finding, not a side issue}
\label{sec:5.3}
% STUB: The location sub-score's empirical near-saturation (§3.6) is the
% binding constraint on confirmatory power. This is a property of the
% as-implemented T2 rubric, not of the dataset or models. Scaling n from
% 30 to 50 or 100 would not change the saturation rate; it would only
% multiply tied paired differences. The follow-up should refine the
% location sub-score or supplement with a continuous T2 metric BEFORE
% expanding n. Methodological finding with re-use value beyond this study.

\subsection{Implications for the long-term Lumen vision}
\label{sec:5.4}
% STUB: Constitution v0.2.2 §3 specifies a reassessment threshold for
% the canonical-core-over-surface-text bet: an outcome of C1+≥C4 on all
% primary metrics across both confirmatory models would have triggered
% formal reassessment. The observed outcome — C4>C1+ in direction with
% moderate effect size and a CI excluding zero, failing Holm — does NOT
% trigger reassessment, nor confirmatory support. The bet's empirical
% motivation remains intact at the level of point estimates and weakened
% at the level of statistical confirmation; the project continues at the
% architectural level pending T1/T3 collection, rubric refinement, and a
% properly-powered confirmatory re-test.
